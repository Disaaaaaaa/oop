Unit Overview

Object-Oriented Programming

OOP is a programming paradigm that organises code around objects rather than actions, and data rather than logic. Python supports OOP natively — every value in Python is an object.

This unit maps to Issue 1 of the Grade 11 curriculum (July 2021, Version 5) and covers three major topic groups.

The 4 Pillars of OOP

Encapsulation

Bundling data (attributes) and behaviour (methods) into a single unit (class), hiding internal state.

Inheritance

A child class inherits attributes and methods from a parent class, enabling code reuse and hierarchy.

Polymorphism

The same method name behaves differently in different classes, allowing flexible and extensible code.

Abstraction

Exposing only essential features and hiding implementation details from the user of a class.

Classes & Instances — LO 11.4.1.1

A class is a blueprint; an instance (object) is a concrete realisation created from that blueprint.

class Car:                      # class declaration
    brand = "Unknown"            # class attribute (shared)

my_car = Car()                   # instance creation
print(my_car.brand)             # → Unknown

Methods — LO 11.4.1.2

Methods are functions defined inside a class. The first parameter is always self, which refers to the current instance.

class Car:
    def honk(self):
        print("Beep beep!")

my_car = Car()
my_car.honk()                   # → Beep beep!

The __init__ Method — LO 11.4.1.3

__init__ is the constructor: it runs automatically when an instance is created and sets default properties.

class Car:
    def __init__(self, brand, speed=0):
        self.brand = brand          # instance attribute
        self.speed = speed

    def accelerate(self, amount):
        self.speed += amount
        print(f"{self.brand} now at {self.speed} km/h")

bmw = Car("BMW")
bmw.accelerate(50)              # → BMW now at 50 km/h

Class Hierarchy & Inheritance — LOs 11.4.1.4 / 11.4.2.2

Use parentheses to specify the parent class. The child inherits all methods and attributes; super() calls the parent's __init__.

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return "..."

class Dog(Animal):              # Dog inherits Animal
    def speak(self):            # method override
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} says Meow!"

animals = [Dog("Rex"), Cat("Whiskers")]
for a in animals:
    print(a.speak())             # polymorphism in action

Polymorphism — LO 11.4.2.1

Polymorphism means "many forms". The same method name (speak(), area()) behaves differently depending on which class's object it is called on.

class Shape:
    def area(self): return 0

class Circle(Shape):
    def __init__(self, r): self.r = r
    def area(self): return 3.14159 * self.r ** 2

class Rectangle(Shape):
    def __init__(self, w, h): self.w, self.h = w, h
    def area(self): return self.w * self.h

shapes = [Circle(5), Rectangle(4, 6)]
for s in shapes:
    print(f"Area: {s.area():.2f}")   # → 78.54, 24.00
💡

Key insight: The loop doesn't need to know whether it's a Circle or Rectangle — it just calls .area() and polymorphism handles the rest. This is the power of OOP for decomposing complex, applied problems (LO 11.4.3.1).
